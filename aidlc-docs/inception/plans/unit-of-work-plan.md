### Q1: ユニット分割の基本方針

チームは実質2名（+画像担当が海外出張中でほぼ不在）で2週間の開発です。この状況でユニットをどう分割しますか？

A) 3ユニット（フロントエンド / バックエンド / インフラ）— 明確に責務分離
B) 3ユニット（フロントエンド / バックエンド / アセット・イラスト）— 画像担当に独立ユニットを割り当て
C) 2ユニット（フロントエンド / バックエンド + インフラ）— インフラはバックエンド担当が兼務
D) その他（具体的に記入）

[Answer]: D 3ユニット（フロントエンド / バックエンド + インフラ / アセット・イラスト）

### Q2: 画像担当（海外出張中）の扱い

画像担当が海外出張で不在がちです。画像素材はどう管理しますか？

A) 独立ユニットとして明確に分離（アセットユニット）。進捗追跡・成果物管理が明確に
B) バックログ的に扱う（必要な時にタイムリーに作成依頼）。ユニットとしては定義しない
C) プレースホルダーで開発し、最終日に一括差し替え

[Answer]: A

### Q6: CONSTRUCTION phaseの進め方

2つのメインユニット（フロントエンド / バックエンド）を並列開発する場合、Functional DesignやCode Generationも並列で進めますか？

A) 並列（FE/BE両ユニットのFunctional Design → 両ユニットのCode Generationを同時進行）
B) 直列（BE先行。API仕様を固めてからFE）
C) BEのコアAPIを先行→並列（BEの最小骨格が動いたらFEを並列開始）

[Answer]: A

### Q7: ユーザーストーリーのユニット割り当て

各ストーリーが複数ユニットにまたがる場合、どう扱いますか？

A) ストーリーを「FE担当分」「BE担当分」に分解して記録（細かく）
B) ストーリーの主責務がある側のユニットに割り当て、もう一方はサポート（シンプル）
C) 画面・API・インフラの3軸で横断的に管理

[Answer]: C

### Q8: インフラ（CDK）の位置づけ

インフラはバックエンド担当が兼務する想定ですが、ユニットとして分けますか？

A) バックエンドユニットに統合（1ユニットで管理）
