## Question 2の解説

LINE連携には複数の技術要素があり、それぞれ役割が異なります：

| 技術要素 | 役割 |
|---|---|
| **LIFF（LINE Front-end Framework）** | LINEアプリ内でWebアプリを開くための仕組み。ユーザー認証（LINEログイン）もこれで行う。アプリのUI自体はReactで作り、LINEアプリ内のブラウザで表示される |
| **LINE Messaging API** | LINEのトーク画面でボットとしてメッセージを送受信する仕組み。おつかいの会話（ナマケモノとのやりとり）はこれで実現する |
| **Push通知（Messaging APIの一部）** | ユーザーがアプリを開いていなくても、LINEトークにメッセージを送る仕組み。「森に新しい仲間が来ているよ！」等の通知に使う |

**選択肢の違い：**

- **A) LIFFアプリとしてフル実装** = 全てをLINEアプリ内で完結させる設計。森の画面もLINE内で開き、おつかいトークもLINE内で行い、通知もLINEに届く。ユーザーはLINEだけで全ての体験ができる。
- **B) LIFFで認証のみ + Push API + Messaging API** = 実質的にAと同じ構成。技術的にはAもBも同じAPIを使うため、違いはありません。（質問の選択肢設計が不適切でした。申し訳ありません。）
