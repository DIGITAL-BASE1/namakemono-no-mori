# ユーザーストーリー

## スコープ凡例
- 🟢 **MVP v1** — 第一次審査（予選）で実装
- 🔵 **MVP v2** — 決勝戦で追加実装

---

## Epic 1: オンボーディング・初回体験

### US-01: 絵本オンボーディング 🟢 MVP v1
**As a** 初めてアプリを開いたユーザー（田中ユウキ）
**I want to** 絵本形式の短い導入で世界観を理解したい
**So that** 操作説明なしで「ナマケモノの森」の雰囲気に入れる

**受け入れ基準:**
```gherkin
Given ユーザーが初めてLIFFアプリを開いた
When オンボーディングが表示される
Then 3ページの絵本形式で世界観が紹介される
And 各ページはタップ/スワイプで進められる
And 操作説明や義務感のある文言は含まれない
```

**デモシナリオ対応:** Day 1 初回起動

---

### US-02: 初回スワイプ体験（チュートリアル） 🟢 MVP v1
**As a** 初めてアプリを開いたユーザー（田中ユウキ）
**I want to** スワイプ操作を自然に覚えたい
**So that** 迷わずダメ報告ができるようになる

**受け入れ基準:**
```gherkin
Given オンボーディング絵本が完了した
When スワイプ体験画面が表示される
Then 5枚のダメカードが表示される
And 1枚目は必ず「ねむねむ」系カードで、操作チュートリアルとして「やった」方向にのみスワイプ可能
And 2枚目は必ず「もぐもぐ」系カードで、「やった」方向にのみスワイプ可能
And 3枚目以降は自由にスワイプできる
And スワイプごとにナマケコインが獲得される
```

**デモシナリオ対応:** Day 1 初回起動（スワイプ体験）

---

### US-03: 初回ナマケモノ召喚 🟢 MVP v1
**As a** 初回スワイプを完了したユーザー（田中ユウキ）
**I want to** 最初のナマケモノが森にやってくるのを見たい
**So that** 「自分のダメでナマケモノが来た」という体験の核心を理解できる

**受け入れ基準:**
```gherkin
Given 初回スワイプ体験で「ねむねむ」カテゴリを「やった」にスワイプした（強制）
When スワイプ体験が完了する
Then ネムリンが森に出現する演出が再生される
And 図鑑にネムリンが登録される（シルエット→実体）
And ナマケコインの獲得合計が表示される
And モグモグは翌朝に出現する（「もぐもぐ」を「やった」にスワイプ済みだが、即時ではなく翌朝）
```

**デモシナリオ対応:** Day 1 初回起動（ネムリン召喚）

## Epic 2: スワイプ報告

### US-04: 通常スワイプ報告 🟢 MVP v1
**As a** 毎日アプリを使うユーザー（田中ユウキ）
**I want to** 今日の自分のダメをスワイプで気軽に報告したい
**So that** ナマケコインを獲得し、ナマケモノの出現条件を進められる

**受け入れ基準:**
```gherkin
Given ユーザーがスワイプ報告画面を開いた
When 全30種・8カテゴリからランダムに10枚のダメカードが表示される
Then 各カードを左右にスワイプして「やった/やってない」を選択できる
And 「やった」1枚につき5NMKが獲得される
And スワイプ完了後に出現条件の進行状況がほのめかされる
And 1日1回のみ実行可能
```

**デモシナリオ対応:** Day 2 夜

---

### US-05: 途中離脱ボーナス 🟢 MVP v1
**As a** 疲れてスワイプを最後まで続けられないユーザー（佐藤ミホ）
**I want to** 途中でやめても報われたい
**So that** 「全部やらなきゃ」というプレッシャーを感じない

**受け入れ基準:**
```gherkin
Given ユーザーがスワイプ報告を開始した
When 10枚中N枚目で×ボタンを押して離脱する
Then 残り(10-N)枚 × 3NMKの途中離脱ボーナスが獲得される
And 途中離脱回数がカウントされる（メンドクサウルス出現条件用）

Given ユーザーがスワイプ報告画面を開いた
When 1枚もスワイプせずに閉じる
Then 20NMK（究極のナマケコイン）が獲得される
```

**デモシナリオ対応:** Day 1 初回起動（キラーシーン）

## Epic 3: ナマケコイン経済圏

### US-06: ショップでアイテム購入 🟢 MVP v1
**As a** ナマケコインを貯めたユーザー（田中ユウキ）
**I want to** ショップでアイテムを購入したい
**So that** ナマケモノに贈って親密度を上げられる

**受け入れ基準:**
```gherkin
Given ユーザーがショップ画面を開いた
When アイテム一覧が表示される
Then おやつ(10〜30NMK)、おもちゃ(50〜100NMK)、特別な贈り物(200〜500NMK)のカテゴリが表示される
And 所持コインが足りるアイテムのみ購入可能
And 購入するとコインが減り、アイテムが所持品に追加される
```

**デモシナリオ対応:** Day 1 初回起動（ショップ体験）

---

### US-07: 日次コイン収穫 🟢 MVP v1
**As a** 森にナマケモノがいるユーザー（佐藤ミホ）
**I want to** ナマケモノが生んだコインを収穫したい
**So that** 放置していてもコインが貯まる実感を得られる

**受け入れ基準:**
```gherkin
Given 森に1匹以上のナマケモノが住んでいる
And 前回収穫から24時間以上経過している
When ユーザーがアプリを開く
Then 各ナマケモノが生んだコイン（5〜15NMK/匹）が収穫可能になる
And 親密度が高いナマケモノほどボーナスコインが加算される
And 収穫演出が表示される
```

**デモシナリオ対応:** Day 2 朝（2匹分の収穫）

---

### US-08: 放置ボーナス蓄積 🟢 MVP v1
**As a** 数日間アプリを開かなかったユーザー（佐藤ミホ）
**I want to** 放置した日数分のボーナスをまとめて受け取りたい
**So that** アプリを開かなかったことに罪悪感を感じない

**受け入れ基準:**
```gherkin
Given ユーザーがN日間スワイプ報告をしなかった
When 次にアプリを開く
Then N日 × 10NMKの放置ボーナスがまとめて付与される
And 「○日間のサボりボーナス！」と表示される
```

**デモシナリオ対応:** —（1.5日体験では発生しないが、仕組みとして実装）

## Epic 4: ナマケモノ・コレクション

### US-09: ナマケモノ出現・図鑑登録 🟢 MVP v1
**As a** ダメ報告を続けているユーザー（木村ソウタ）
**I want to** 新しいナマケモノが森に来たことに気づきたい
**So that** 図鑑を埋める喜びを感じられる

**受け入れ基準:**
```gherkin
Given ユーザーのダメ報告がナマケモノの出現条件を満たした
When 次にアプリを開く（翌日以降）
Then 新しいナマケモノが森に出現する演出が再生される
And 図鑑のシルエットが実体に変わる
And ヒント2（やや具体的）が解放される（条件50%達成時）

Given 図鑑画面を開いた
When 未発見のナマケモノを確認する
Then シルエットとヒント1（曖昧）が表示される
And 16体分の枠が表示される（実装5体 + シルエット11体）
```

**デモシナリオ対応:** Day 2 朝（モグモグ出現）、Day 3 朝（条件次第のナマケモノ）

## Epic 5: 親密度・手紙

### US-11: アイテム贈呈・親密度UP 🟢 MVP v1
**As a** ナマケモノと仲良くなりたいユーザー（田中ユウキ）
**I want to** アイテムをナマケモノに贈って親密度を上げたい
**So that** 手紙やおつかいが解放される

**受け入れ基準:**
```gherkin
Given ユーザーが所持品にアイテムを持っている
When ナマケモノを選択してアイテムを贈る
Then 親密度が上昇する（おやつ=小、おもちゃ=中、特別な贈り物=大）
And 親密度UP演出が表示される
And 現在の親密度レベル（★1〜★5）が確認できる
```

**デモシナリオ対応:** Day 1 初回起動（贈り物体験）

## Epic 6: 通知

### US-13: 夜のスワイプ通知 🟢 MVP v1
**As a** アプリを閉じているユーザー（田中ユウキ）
**I want to** ナマケモノからの通知でスワイプ報告を思い出したい
**So that** 毎日のループが自然に回る

**受け入れ基準:**
```gherkin
Given デフォルト通知時刻（21:00）になった
And 2日目以降である（初日は送らない）
When LINE Push通知が送信される
Then 森にいるナマケモノからランダムで1匹が選ばれる
And そのキャラ固有のセリフで通知が届く（例：ネムリン「もう寝た？ぼくはもう寝るよ〜」）
And 通知タップでスワイプ報告画面に遷移する
```

**デモシナリオ対応:** Day 2 夜

---

### US-14: 朝の到着通知 🟢 MVP v1
**As a** 新しいナマケモノの出現条件を満たしたユーザー（木村ソウタ）
**I want to** 朝に「新しい仲間が来た」と知りたい
**So that** アプリを開く動機が生まれる

## Epic 7: おつかい（AIエージェント）

### US-15: おつかい受信 🟢 MVP v1
**As a** 親密度MAXに到達したユーザー（木村ソウタ）
**I want to** ナマケモノからLINEでおつかいを頼まれたい
**So that** 現実世界で小さな行動をするきっかけになる

**受け入れ基準:**
```gherkin
Given ナマケモノとの親密度が★5（MAX）に到達している
And 前回のおつかいから2〜3日経過している
When おつかいが配信される
Then LINEトークにナマケモノのキャラクターでおつかいメッセージが届く
And おつかい内容はAI（Bedrock）がパーソナライズして生成する
And 天気・曜日・履歴を考慮したガードレールが適用される
And 配信時刻は10:00/13:00/16:00のいずれか（日替わりランダム）
And おつかいレベル（1〜4）はユーザーの達成状況に応じて調整される
```

**デモシナリオ対応:** 3週間後の世界（プレゼンで事前セットアップ状態を見せる）

---

### US-16: おつかい報告・AI応答 🟢 MVP v1
**As a** おつかいを実行したユーザー（木村ソウタ）
**I want to** ナマケモノに報告して反応を見たい
**So that** ナマケモノとの会話を楽しめる

**受け入れ基準:**
```gherkin
Given ユーザーがおつかいを受信している
When LINEトークに写真またはテキストで報告する
Then ナマケモノがAI（Bedrock）で応答する（キャラクターのトーンで）
And 2〜3往復の短い会話が行われる（最大10往復まで）
And AIが達成/未達成を判定する
And 未達成でもペナルティなし（「まあいいや。気が向いたらまた今度ね。」）
And プロンプトインジェクション対策が適用される（NFR-007）
```

**デモシナリオ対応:** 3週間後の世界（おつかい会話デモ）

---

### US-17: 森の日記 🟢 MVP v1
**As a** おつかいを報告したユーザー（木村ソウタ）
**I want to** おつかいの記録がアプリに残っていてほしい
**So that** 振り返ると「自分が動いた軌跡」が見える

**受け入れ基準:**
```gherkin
Given ユーザーがおつかいを報告し、AI応答が完了した
And AIが達成と判定した
When 森の日記画面を開く
Then おつかいの内容と報告がナマケモノの言葉で記録されている
And 日記テキストはBedrockが会話ログからキャラクタートーンで要約生成したものである
And 日付とナマケモノ名が表示される
And 過去の日記を時系列で閲覧できる

Given おつかいが未達成と判定された
When 森の日記画面を開く
Then そのおつかいの記録は日記に残っていない
```

**デモシナリオ対応:** 3週間後の世界（森の日記に軌跡が残っている）

## ストーリー依存関係

```
US-01 (絵本) → US-02 (初回スワイプ) → US-03 (初回召喚)
                                              ↓
US-04 (通常スワイプ) ←────────────────────────┘
US-05 (途中離脱) ←─── US-04と並行
      ↓
US-09 (ナマケモノ出現) ← US-04の繰り返し
      ↓
US-07 (日次収穫) ← US-09（ナマケモノが1匹以上）
US-06 (ショップ) ← US-07 or US-04（コインがある）
      ↓
US-11 (贈呈・親密度) ← US-06（アイテムがある）
      ↓
US-12 (手紙) ← US-11（親密度★3）
      ↓
US-15 (おつかい受信) ← US-11（親密度★5）
US-16 (おつかい報告) ← US-15
US-17 (森の日記) ← US-16

US-08 (放置ボーナス) ← 独立（ログインしない日に蓄積）
US-10 (つぶやき) ← US-09（ナマケモノが1匹以上）
US-13 (夜の通知) ← US-09（2日目以降）
US-14 (朝の通知) ← US-09（出現条件達成時）
```
