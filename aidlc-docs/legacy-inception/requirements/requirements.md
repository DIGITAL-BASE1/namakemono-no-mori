# 要件定義書：🦥ナマケモノの森

---

## プロジェクト概要

### サービス名
**ナマケモノの森 — サボるほど豊かになる、ナマケだらけの経済圏**

### テーマ
AWS主催ハッカソン「人をダメにするサービス」

### コアメッセージ
> **「サボるほど豊かになる、ナマケだらけの経済圏へようこそ。」**

### コアロジック
```
社会の問題：
  SNS・自己啓発文化が「理想の自分」のハードルを際限なく上げている
  → ついつい見続けてしまうSNSは、人をダメにする一方で、
    他人のキラキラした姿を突きつけられ、自己否定を加速させる
  → 「生産性」「成長」「自己投資」が美徳とされ、休むことに罪悪感がある
  → 結果、普通に生きているだけで他者と比較し「他人は輝いている、自分はダメだ」と感じる人がいる

このサービスの提案：
  ダメな自分を受け入れよう
  サボることに価値を与えよう
  怠惰を通貨にしよう
```

**核心は「価値の逆転」。** ダメを治すのではない。ダメに価値を与える。怠惰を通貨にする。サボるほど森が豊かになり、ナマケモノたちが集まってくる。

**SNSとの対比**: SNSは人をダメにしながら、同時に自己否定を残す。このサービスも人をダメにするが、残すのは自己肯定。同じ「人をダメにする」でも、出口が真逆。

### 一言コンセプト
> **「自分のダメを報告すると、ナマケモノが勝手に集まってくる。どのダメで誰が来るかは、やってみないとわからない。」**

### エレベーターピッチ（30秒）
> 日常の「ダメだったこと」を報告するだけ。すると、あなたのダメに共感したナマケモノたちが、いつの間にか森に住み着いている。どのナマケモノが来るかは、ダメの種類や組み合わせ次第。条件はヒントだけ。図鑑を埋めたくなる。ダメを報告するほど森が賑やかになり、ナマケモノたちと過ごすうちに「ダメな自分も悪くないな」と思えてくる。

## MVPスコープ定義

### 設計方針
> **1.5日分の体験を完全に実装し、コアループが回る実感を審査員に届ける。**
> 1.5日体験に不要な要素は実装せず、プレゼンで将来ビジョンとして語る。

### 審査形式の前提
- **審査員はプレゼンを見るだけ**。サービスを手元で操作することはない
- 1.5日体験の各シーンを**短い動画（数秒〜十数秒）としてスライドに埋め込む**
- 動くプロダクトはAWSにデプロイして構築する（動画撮影用＋デモ完成度の評価対象）
- 一般公開はしない

### 実装戦略への影響
| 方針 | 理由 |
|---|---|
| **ハッピーパス最優先** | 動画で見せる導線が完璧に動けばよい |
| **エッジケース処理は最小限** | 審査員が予想外の操作をすることがない |
| **データの事前準備OK** | Day2朝の状態をセットアップして動画撮影できる |
| **演出への投資** | 堅牢性に使う時間を、見た目・演出の磨き込みに回せる |

## コアゲームデザイン

### コアループ

```
ダメをスワイプ報告（or 途中離脱 or 放置）
        |
        v
  ナマケコイン獲得 + ナマケモノ出現条件の進行
        |
        v
  条件を満たすと、ナマケモノが森に住み着く
        |
        v
  住み着いたナマケモノが毎日一緒に怠けてナマケコインを生む（1日1回収穫）
        |
        v
  ナマケコインでショップからアイテムを購入
        |
        v
  アイテムをナマケモノに贈る → 親密度UP
        |
        v
  親密度が上がると贈り物・手紙・特別な反応が解放
        |
        v
  もっとナマケモノを集めたい（コイン収穫量UP）→ もっとダメを報告したくなる
```

### ループが回り続ける理由

| 動機 | 仕組み |
|---|---|
| **収集欲** | 図鑑を埋めたい → ダメ報告で出現条件を進める |
| **育成欲** | ナマケモノとの親密度を上げたい → コインでアイテムを買って贈る |
| **経済欲** | コイン収穫量を増やしたい → ナマケモノを増やす → ダメ報告 |
| **発見欲** | ナマケモノ同士の化学反応を見たい → 新しい組み合わせを試す |
| **愛着** | 育てたナマケモノへの愛着 → 毎日森を覗きたくなる |

### 設計原則

1. **放置もアリ**: 何もしなくても森は少しずつ豊かになる。放置ボーナスあり。ユーザーに勤勉さを求めない
2. **条件非公開**: どのダメでどのナマケモノが来るかは試行錯誤。ヒントのみ
3. **失敗がない**: ペナルティなし。ダメを報告するだけで良いことしか起きない
4. **押し付けない前向きさ**: 「頑張れ」とは絶対に言わない。自然に前向きになる仕組み

## ナマケコイン（通貨システム：単位NMK）

### 獲得源

| 獲得方法 | コイン量（目安：付与コイン数はゲームバランス上調整必須） |
|---|---|
| スワイプで「やった（さぼった）」1枚 | 5コイン |
| スワイプ途中離脱（怠惰ポイント） | 残り枚数 × 3コイン |
| 1枚もスワイプせず閉じる | 20コイン（究極の怠惰。スワイプすら面倒） |
| 放置ボーナス（申告しなかった日） | 1日あたり10コイン蓄積 |
| **ナマケモノの日次収穫** | ナマケモノ1匹あたり5〜15コイン/日（種類による） |

### 日次収穫の仕組み

> **森にいるナマケモノが毎日一緒に怠けてくれることで、ナマケコインが生まれる。**

- 1日1回、ログイン時にまとめて収穫
- ナマケモノの数が多いほど収穫量が増える
- レア種・伝説種ほど1匹あたりの収穫量が多い
- 親密度が高いナマケモノはボーナスコインを生む

### 使い道：ショップ

| カテゴリ | アイテム例 | 価格帯 | 効果 |
|---|---|---|---|
| **おやつ** | ナマケモノクッキー、ぐうたらプリン | 10〜30コイン | 親密度+小 |
| **おもちゃ** | ハンモック、もこもこクッション | 50〜100コイン | 親密度+中 |
| **特別な贈り物** | 王冠、虹色マフラー | 200〜500コイン | 親密度+大 |

### 獲得と消費のバランス

| 頻度 | 体験 |
|---|---|
| **毎日** | おやつ程度の小さな買い物ができる |
| **2〜3日** | おもちゃ級のアイテムが買える |
| **1〜2週間** | 特別な贈り物に手が届く |

## ダメ報告システム

### 設計思想
> **申告は気楽に、途中でやめても報われる。**
> 全部やらなくていい。やった（さぼった）分だけ良いことが起き、やめた分も良いことが起きる。申告を途中でやめる行為すらダメとして報われる。

### メインUI：スワイプ方式

ダメの種類が書かれたカードが次々と表示され、ユーザーは「やった（さぼった）/やってない」をスワイプで仕分けるだけ。

```
+---------------------------+
|                           |
|    🛏️ ねむねむ            |
|                           |
|    「二度寝した」          |
|                           |
|                           |
|   ← やってない  やった →   |
|                           |
+---------------------------+
       [ 1 / 10 ]
```

**カード枚数**: 約10枚/回
- カードはカテゴリアイコン + ダメの内容で構成
- カテゴリはアイコン+ゆるい名前でゆるく表示
- カードの内容は固定プリセット
- 全30種類の中からランダムで10枚（各カテゴリから最低1枚）

### 途中離脱ボーナス（核心的な仕掛け）

> **スワイプを途中でやめると「ナマケコイン」が付与される。**

- 10枚中3枚でやめた → 「7枚分のナマケコイン」獲得
- 1枚もスワイプせずに閉じた → 「究極のナマケコイン」獲得
- **途中離脱でしか解放されないナマケモノがいる**
  - 例：「メンドクサウルス」— スワイプを5回以上途中でやめると出現
  - 例：「ゼロスワイプン」— 一度もスワイプせずに閉じるを3回で出現

**なぜこれが重要か**:
- 「申告をサボる」ことすら報われる → 摩擦ゼロ原則の究極形
- ユーザーは「全部スワイプしなきゃ」というプレッシャーを感じない
- 「途中でやめた方が得かも？」という逆転の判断が生まれる
- サービスの哲学「怠惰に価値を与える」が体験レベルで実現される

### 申告タイミング

| タイミング | 説明 |
|---|---|
| **夜のスワイプ通知** | 毎晩（時間はユーザー設定可）にナマケモノ固有セリフで通知。初日は送らない |
| **自由タイミング** | 通知のタイミングでなくても、好きな時に1日1回申告可能 |
| **通知を無視** | 無視しても放置ボーナスが溜まる。ペナルティなし |

### 申告後のフィードバック

| フィードバック | 説明 |
|---|---|
| **ナマケモノの反応** | 森のナマケモノがリアクション（喜ぶ、共感する、一緒にゴロゴロする） |
| **条件進捗のほのめかし** | 「森に誰かの気配がしますよ…」「足跡が増えている…」 |

### 申告しなかった日の扱い

**放置ボーナス**:
- 申告しなかった日数に応じてボーナスが蓄積
- 次回ログイン時に「○日間のサボりボーナス！」として受け取り
- 長期放置でしか出現しないナマケモノの条件にもカウント

### ダメカテゴリ

カテゴリはアイコン+ゆるい名前で表示。スワイプカードに付与：

| アイコン | カテゴリ | カード内容 |
|---|---|---|

## ナマケモノ図鑑

### 規模
**30体以上**（ノーマル種・レア種・季節限定種・伝説種）

### 出現条件の設計ルール

1. **自然に達成できる**: 普通に生活をしていれば、いつの間にか条件を満たす。ダメだと思っていることは実は普通だよって肯定してあげる
2. **狙いすぎない**: 特定のダメを「頑張って」する必要がない（本末転倒）
3. **組み合わせの妙**: 複数のダメの種類・回数の組み合わせで条件が決まる
4. **時間軸**: 一部は「○日連続」「○週間以内に」など時間要素を含む
5. **季節・イベント**: 季節限定ナマケモノで長期的な楽しみを提供
6. **究極の放置報酬**: アプリを開かないことが条件のナマケモノがいる

### ヒントシステム（2段階）

| 段階 | 表示タイミング | 例 |
|---|---|---|
| **ヒント1（曖昧）** | 最初から図鑑に表示 | 「眠りを愛する者のもとに現れるらしい…」 |
| **ヒント2（やや具体的）** | 条件の50%達成時 | 「あと少し…眠りの回数が足りないようだ」 |

- 図鑑の未発見ナマケモノは**シルエット**で表示
- ヒントは**古文書風**の文体
- 条件達成が近づくと、森に**足跡**が現れる

## 親密度システム

### 仕組み

各ナマケモノには**親密度**パラメータがある。親密度を上げることで、ナマケモノとの関係が深まる。

### ナマケモノの個性

各ナマケモノには**好きなもの**と**きらいなもの**が設定されている：

| ナマケモノ | 好きなもの | きらいなもの |
|---|---|---|
| ネムリン | もこもこクッション、ホットミルク | 目覚まし時計 |
| モグモグ | ぐうたらプリン、ポテチ | ダイエット食品 |
| アトデーノ | のんびりお茶 | スケジュール帳 |
| スクロン | キラキラステッカー、充電器 | 紙の本 |

※ 全キャラクターの個性は詳細設計フェーズで設定

### 親密度の上げ方

- ショップで買ったアイテムをナマケモノに贈る
- **好きなもの**を贈ると親密度が大きく上がる
- **きらいなもの**を贈ると親密度が下がる（ただしペナルティは軽微）
- 好き嫌いは明示されない。キャラクターの見た目や性格から推測する

### 親密度で解放される要素

| 親密度レベル | 解放要素 |
|---|---|
| ★★★☆☆ | ナマケモノからの手紙 |
| ★★★★★ | レアな贈り物（コレクションアイテム） |

### 贈り物（第2の収集要素）

- 親密度が最大になったナマケモノは、時々レアな贈り物をくれる
- 贈り物は「贈り物コレクション」に記録される
- レアなナマケモノほどレアな贈り物
- 全贈り物コンプリートも長期目標の一つ

## 図鑑・コレクション

3つの収集要素がループを回し続ける：

| 図鑑 | 内容 | 集め方 |
|---|---|---|
| **ナマケモノ図鑑** | 全ナマケモノのシルエット・ヒント・発見記録 | ダメ報告で出現条件を満たす |
| **手紙コレクション** | ナマケモノからもらった手紙をすべて保管 | 親密度を上げて手紙を受け取る |
| **贈り物コレクション** | ナマケモノからもらったレアアイテム | 親密度MAXで贈り物を受け取る |

> 人からもらった手紙って捨てないよね。ナマケモノからの手紙も、全部残る。
> 読み返すたびに、そのナマケモノとの思い出が蘇る。エモい。

## 社会的意義

### このサービスが届けたい価値

1. **「ダメでいい」という許可**
   - 「生産性」「成長」が美徳の社会で、休むことに罪悪感を感じている人に
   - ゲーム体験を通じて「ダメな自分も悪くない」と自然に思える場所を作る

2. **自己肯定感の逆転**
   - ダメを報告するほど森が豊かになる = ダメな自分に価値がある
   - ナマケモノたちが自分のダメに共感してくれる = 一人じゃない
   - 休んだ日・サボった日を「失敗した日」として終わらせず、森の成長やナマケコインという資産に変える
 
このサービスは、ユーザーが自分を責めがちな出来事を、ゲーム内で価値あるものとして扱う。結果として、休むことやサボることへの罪悪感が少し軽くなり、「ダメな自分も悪くない」と自然に感じられる体験を目指す。ダメであることを肯定することで、「人をダメにするサービス」として成り立つ。

### 通知システム

#### 通知の種類

| 種類 | タイミング | 内容 | 遷移先 |
|---|---|---|---|
| **朝の到着通知** | 新しいナマケモノが来た朝 | 「森に新しい仲間が来ているよ！見に行こう🦥」 | 森画面 |
| **夜のスワイプ通知** | 毎晩（ユーザー設定時刻） | ナマケモノ固有のセリフ | スワイプ報告画面 |

#### 通知ルール

- **初日**: 夜の通知を送らない（オンボーディングでスワイプ済み。翌朝の到着通知で再エンゲージ）
- **2日目以降**: 夜の通知を毎晩送る
- **朝の到着通知**: 新しいナマケモノが来た朝のみ送る（毎朝ではない）
- **夜の通知セリフ**: 森にいるナマケモノからランダムで1匹選出。そのキャラ固有のセリフ
- ナマケモノが増えるほど、通知のバリエーションが増える
- 各ナマケモノは複数のセリフパターンを持つ
- 親密度が高いナマケモノほど選ばれやすい（任意設定）
- 通知をタップするとスワイプ報告画面へ直接遷移
- 通知を無視してもペナルティなし。放置ボーナスが溜まる

#### MVP v1での実装
- **LINE Messaging APIによる本物のPush通知を実装**（LIFFを選んだ最大の理由）
- EventBridgeスケジューラーで夜の通知・朝の到着チェックを自動実行
- 開発用手動トリガーAPI（動画撮影時にタイミングを制御するため）も併設
- 動画撮影時は手動トリガーで好きなタイミングに通知を発火可能
