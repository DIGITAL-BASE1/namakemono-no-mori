# アイデア精錬：「ナマケモノ経済圏 × RPG」

---

## コンセプト統合

### 一言で言うと
> **「ダメな自分を報告するほど、RPGのキャラが強くなる」**

### エレベーターピッチ（30秒）
> 現代人は「ダメな自分」に罪悪感を感じすぎている。このサービスは、日常の「ダメだったこと」を報告するとAIがそれを査定し、「ナマケコイン」に変換。コインでRPGキャラクターのステータスを上げたり、スキルや装備を手に入れたりできる。ダメであればあるほどキャラが強くなる。「ダメ」が「成長の糧」に変わる逆転のRPG。

---

## サービス設計の深さ：4層ストーリー

### 第1層：社会課題への洞察
**「なぜ現代人はダメだと感じるのか」**

- SNS・自己啓発文化が「理想の自分」のハードルを際限なく上げている
- 「生産性」「成長」「自己投資」が美徳とされ、休むことに罪悪感がある
- 結果、普通に生きているだけで「ダメだ」と感じる人が増加
- **このサービスの提案**: 「ダメ」を罰するのではなく、「ダメ」を資源として活用する社会の仕組み

### 第2層：ユーザー心理の設計
**「どうすれば本当に肯定されたと感じるか」**

- 単に「大丈夫だよ」と言われても響かない
- **ゲーミフィケーションの力**: ダメな行動が「キャラの成長」という目に見える成果に変わることで、脳の報酬系が「ダメ = 良いこと」と再学習する
- **RPGのメタファー**: 現実世界では「ダメ」でも、ゲーム世界では「経験値」。この二重構造が心理的安全性を生む
- **キャラクターへの投影**: 自分のダメで育ったキャラクターは、自分のダメの集大成。それが強くなるということは、ダメな自分にも価値があるということ

### 第3層：技術設計の深さ
**「AIとクラウドだからこそ成立する仕組み」**

- **AI査定**: ダメの内容を理解し、適切なコイン量・ステータス種別を判定（単なるランダムではない）
- **パーソナライズ**: ユーザーの「ダメ傾向」を学習し、キャラクターの成長方向に反映
- **生成AI**: ダメに応じたユニークなスキル名・装備名・ストーリーを動的生成
- **スケーラブル**: サーバーレスアーキテクチャで低コスト運用

### 第4層：ビジネスモデルの深さ
**「持続可能なサービスとしての設計」**

- **基本無料**: ダメ報告とキャラ育成は無料
- **マネタイズ候補**:
  - プレミアムキャラクタースキン
  - 特別なダメカテゴリ（レアスキル解放）
  - 企業向け：従業員のメンタルヘルス施策としての法人プラン
- **社会的インパクト**: メンタルヘルス領域への新しいアプローチとして研究・行政との連携可能性

## ゲームデザイン詳細

### キャラクター設計

**基本キャラクター**: ナマケモノ（ユーザーの分身）

**ステータス体系**:

| ステータス | 対応するダメカテゴリ | 例 |
|---|---|---|
| **HP（ヒットポイント）** | 睡眠・休息系のダメ | 二度寝した、昼寝した、10時間寝た |
| **攻撃力** | 欲望に忠実系のダメ | ジャンクフード食べた、衝動買いした |
| **防御力** | 回避・サボり系のダメ | 仕事サボった、誘いを断った、家から出なかった |
| **素早さ** | 先延ばし系のダメ | 締め切りギリギリ、返信を後回し |
| **知力** | 非生産的活動系のダメ | YouTube3時間、SNS無限スクロール |
| **運** | 予想外のダメ | 寝坊で遅刻、うっかりミス |

### スキル・装備システム

**スキル例**:
| スキル名 | 解放条件 | 効果（フレーバー） |
|---|---|---|

| 「無限スクロール」 | SNS3時間以上を3回報告 | 知力+15、素早さ-5 |

| 「明日の自分に託す」 | 先延ばしを5回報告 | 素早さ+30（逆説的） |

### AI査定の仕組み

```
ユーザー入力: 「今日3時間YouTubeを見てしまった」

AI査定結果:
├── カテゴリ判定: 非生産的活動系
├── ダメ度: ★★★☆☆（中級ダメ）
├── 付与コイン: 30ナマケコイン
├── ステータス上昇: 知力+3
├── コメント: 「3時間の動画視聴は立派な情報収集活動です。
│              あなたの知力は確実に上がっています。」
└── 解放チェック: 「無限スクロール」スキルまであと1回
```

### 成長・進化システム

**レベルアップ**:
- 総ステータスが一定値に達するとレベルアップ
- レベルアップ時に称号を獲得（「見習いナマケモノ」→「一人前のナマケモノ」→「ナマケモノマスター」→「伝説のナマケモノ」）

**進化**:
- 特定のステータスバランスで異なる進化形態に分岐
  - HP特化 → 「眠りの守護者」（睡眠系ダメが多い人）
  - 攻撃特化 → 「欲望の戦士」（欲望系ダメが多い人）
  - 知力特化 → 「虚無の賢者」（非生産的活動が多い人）
  - バランス型 → 「万能のダメ人間」（まんべんなくダメな人）

## 技術アーキテクチャ（概要）

```
+-------------------+     +------------------+     +-------------------+
|                   |     |                  |     |                   |
|   フロントエンド    | --> |  API Gateway     | --> |   Lambda          |
|   (React/Next.js) |     |                  |     |   (ダメ査定API)    |
|                   |     +------------------+     +-------------------+
+-------------------+                                      |
        |                                                  v
        |                                        +-------------------+
        |                                        |                   |
        +--------------------------------------> |  Amazon Bedrock   |
                                                 |  (AI査定エンジン)  |
                                                 +-------------------+
                                                          |
                                                          v
                                                 +-------------------+
                                                 |                   |
                                                 |  DynamoDB         |
                                                 |  (ユーザー/キャラ  |
                                                 |   データ)          |
                                                 +-------------------+
```

**主要AWSサービス**:
- **Amazon Bedrock**: ダメ査定AI、スキル名・装備名の動的生成
- **AWS Lambda**: サーバーレスAPI
- **Amazon API Gateway**: RESTful API
- **Amazon DynamoDB**: ユーザーデータ、キャラクターデータ、ダメ履歴
- **Amazon S3 + CloudFront**: フロントエンドホスティング
- **Amazon Cognito**: ユーザー認証（簡易版）

## リスクと対策

| リスク | 影響 | 対策 |
|---|---|---|
| ゲームバランスの調整が難しい | 面白さに直結 | MVP段階ではシンプルな線形成長、後で調整 |
| AI査定の精度 | ユーザー体験に直結 | プロンプトエンジニアリングで対応、フォールバック用意 |
| 2週間でRPG要素の完成度 | デモの質に直結 | コア機能（査定→コイン→ステータス）に集中、装備は最小限 |
| 「ダメを助長する」という批判 | 社会的評価 | 「肯定」であって「推奨」ではないというメッセージング |
