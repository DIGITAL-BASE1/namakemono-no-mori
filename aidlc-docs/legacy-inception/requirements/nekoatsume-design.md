# コンセプト再設計：「ねこあつめ」型ナマケモノ収集ゲーム

---

## なぜ「ねこあつめ」モデルが最適解なのか

### ねこあつめの天才的な設計

| 要素 | ねこあつめ | 本サービスへの応用 |
|---|---|---|
| **入口の低さ** | グッズを置くだけ | ダメを報告するだけ |
| **放置が正義** | 放っておくと猫が来る | 放っておくとナマケモノが来る |
| **発見の喜び** | 「あ、新しい猫がいる！」 | 「あ、新しいナマケモノがいる！」 |
| **条件の非公開** | どのグッズでどの猫が来るか試行錯誤 | どのダメの組み合わせでどのナマケモノが来るか試行錯誤 |
| **図鑑コンプ欲** | 全猫を集めたい | 全ナマケモノを集めたい |
| **SNS共有** | レア猫のスクショ | レアナマケモノのスクショ |
| **ゼロストレス** | 失敗がない、ペナルティがない | ダメを報告するだけ、失敗がない |

### ねこあつめとの決定的な違い（差別化）

| | ねこあつめ | 本サービス |
|---|---|---|
| **入力** | グッズを買って置く（ゲーム内行動） | **自分のダメを報告する（現実の行動）** |
| **意味** | 純粋な癒しゲーム | **自己肯定 + メンタルヘルス** |
| **条件** | グッズの種類 | **ダメの種類・回数・組み合わせ** |
| **深さ** | 猫が来て去る | **ナマケモノが住み着き、関係性が育つ** |
| **社会性** | 個人プレイ | **ダメの共有、牧場の見せ合い** |

> **ねこあつめが「猫を集める癒し」なら、**
> **このサービスは「ダメな自分を集める癒し」。**
> **集めたダメが、愛おしいナマケモノたちに変わる。**

## 新コンセプト

### 一言で言うと
> **「ダメを報告すると、ナマケモノが勝手に集まってくる。どのダメで誰が来るかは、やってみないとわからない。」**

### エレベーターピッチ（30秒）
> 日常の「ダメだったこと」を報告するだけ。すると、あなたのダメに共感したナマケモノたちが、いつの間にか牧場に住み着いている。どのナマケモノが来るかは、ダメの種類や組み合わせ次第。条件はヒントだけ。図鑑を埋めたくなる。ダメを報告するほど牧場が賑やかになり、ナマケモノたちと過ごすうちに「ダメな自分も悪くないな」と思えてくる。ねこあつめならぬ「ナマケモノあつめ」。

## ゲーム設計

### コアループ

```
ダメを報告する
    ↓
（裏で条件判定）
    ↓
条件を満たしていたら...
    ↓
次にアプリを開いた時、新しいナマケモノが牧場にいる！
    ↓
図鑑に登録 → 「次は誰だろう？」 → もっとダメを報告したくなる
```

### 条件設計の哲学

> **条件は「ダメの組み合わせ」で決まる。**
> ユーザーは自分のダメを正直に報告するだけ。
> 狙って特定のダメをする必要はない（それは「頑張る」ことになるから）。
> 自然な生活の中で、いつの間にか条件が満たされている。

### 条件設計のルール

1. **自然に達成できる**: 普通にダメな生活をしていれば、いつの間にか条件を満たす
2. **狙いすぎない**: 特定のダメを「頑張って」する必要がない（それは本末転倒）
3. **組み合わせの妙**: 単一のダメではなく、複数のダメの組み合わせで条件が決まる
4. **時間軸**: 一部は「○日連続」「○週間以内に」など時間要素を含む
5. **季節・イベント**: 季節限定ナマケモノで長期的な楽しみを提供
6. **究極の放置報酬**: アプリを開かないことが条件のナマケモノがいる（最高の皮肉）

## ヒントシステム

### ヒントの段階

| 段階 | 表示タイミング | 例 |
|---|---|---|
| **ヒント1（曖昧）** | 最初から図鑑に表示 | 「眠りを愛する者のもとに現れるらしい…」 |
| **ヒント2（やや具体的）** | 条件の50%達成時 | 「あと少し…眠りの回数が足りないようだ」 |
| **ヒント3（ほぼ答え）** | 条件の80%達成時 | 「二度寝があと1回…」 |

### ヒントの演出
- 図鑑の未発見ナマケモノは**シルエット**で表示
- ヒントは**古文書風**の文体（「〜らしい…」「〜とか…」）
- 条件達成が近づくと、牧場に**足跡**が現れる（「誰かが来そうな気配…」）

## 牧場の設計

### 牧場の成長

| 段階 | ナマケモノ数 | 牧場の様子 |
|---|---|---|
| **はじまりの庭** | 0〜2匹 | 小さな空き地。木が1本 |
| **ナマケモノの丘** | 3〜5匹 | 草原が広がる。ハンモックが出現 |
| **ぐうたらの森** | 6〜10匹 | 木々が増え、小川が流れる |
| **ナマケモノ楽園** | 11匹以上 | 花が咲き乱れ、温泉が湧く |
| **伝説の牧場** | 全種コンプ | 虹がかかり、神々しい光に包まれる |

### 牧場でのナマケモノの行動

ナマケモノたちは牧場で自由に過ごしている：
- 木の上で寝ている
- 他のナマケモノとゴロゴロしている
- おやつを食べている
- ぼーっと空を見ている
- たまに面白いことをしている（レアアクション）

### ナマケモノ同士の関係

- 特定の組み合わせのナマケモノが一緒にいると**特別なイベント**が発生
  - ネムリン + ゴロリーナ → 「究極のお昼寝タイム」（2匹で重なって寝る）
  - モグモグ + カッチャウ → 「爆買いピクニック」（一緒におやつを食べる）
- これも図鑑に記録される（**イベント図鑑**）

## 前向き要素の統合（ねこあつめ型に合わせて再設計）

### 押し付けない前向きさ（v2）

ねこあつめ型では、前向き要素も「発見」の形で提供する：

| 仕組み | 説明 |
|---|---|
| **ナマケモノの一言** | 牧場のナマケモノをタップすると、一言つぶやく。その中に時々前向きな言葉が混ざる。「今日もいい天気だね〜」「お水飲んだ？」「あなたがいるから楽しいよ」 |
| **手紙システム** | 親密度が上がったナマケモノが、時々「手紙」を残す。手紙には、そのナマケモノの物語と、さりげない前向きメッセージが書かれている |
| **牧場の変化** | 牧場が育つこと自体が「自分のダメがこんなに豊かな場所を作った」という肯定 |
| **振り返り機能** | 月末に「今月のダメまとめ」が届く。ダメの量ではなく「こんなにナマケモノが来たよ」という形で |

### 手紙の例（ネムリンからの手紙）

```
ネムリンからの手紙 🌙

ぼく、ネムリンだよ。
あなたの牧場に来てから、毎日よく眠れてるよ。

ぼくね、前はずっと一人で寝てたんだ。
でもここに来てから、みんなと一緒にお昼寝できて嬉しいよ。

あなたも、たまにはゆっくり寝てね。
眠ることは、明日の自分への贈り物だから。

                        ネムリンより
```

## AI査定の再設計（ねこあつめ型）

### 査定の役割の変化

| Before（RPG型） | After（ねこあつめ型） |
|---|---|
| ダメを査定してコイン付与 | ダメを**分類**して条件判定に使う |
| ステータスを上げる | ナマケモノの出現条件をチェック |
| スキル・装備を解放 | 図鑑のヒントを更新 |

### AI査定フロー

```
ユーザー入力: 「今日3時間YouTubeを見てしまった」

AI処理:
├── カテゴリ判定: 動画視聴系
├── 回数カウント: 動画2時間以上 → 累計3回目
├── 条件チェック:
│   ├── スクロン: SNS1時間以上×3 + 動画2時間以上×2 → 動画2回/2 ✓ SNS未達
│   ├── ニジイロナマケ: 全カテゴリ報告 → 動画系 ✓ 他未達
│   └── ...
├── ヒント更新: スクロンのヒント2解放（条件50%達成）
└── レスポンス:
    「3時間の動画視聴、お疲れさまです。
     牧場に誰かの気配がしますよ…？」
```

## ねこあつめとの比較まとめ

| 要素 | ねこあつめ | 本サービス |
|---|---|---|
| 集めるもの | 猫 | ナマケモノ |
| 引き寄せる方法 | グッズを置く | ダメを報告する |

| 課金要素 | にぼし（ゲーム内通貨） | 牧場デコレーション等 |
| **独自要素** | — | 現実のダメが入力、前向き要素、社会的意義 |
