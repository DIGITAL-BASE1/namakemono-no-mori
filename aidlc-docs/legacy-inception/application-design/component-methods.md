# コンポーネントメソッド定義 — ナマケモノの森 MVP v1

---

## 目的
各コンポーネントが提供する機能（インターフェース）を高レベルで定義する。
具体的なメソッドシグネチャ、リクエスト/レスポンススキーマ、ビジネスルールの詳細はCONSTRUCTION phaseで定義。

---

## バックエンド API 機能一覧

### ユーザー管理
| 機能 | 目的 |
|---|---|
| ユーザー登録 | 初回LIFF起動時にLINE IDを取得・保存。Push通知の前提条件 |
| ユーザー状態取得 | コイン残高、ナマケモノ一覧、親密度、図鑑進捗を一括取得 |

### ゲームコア
| 機能 | 目的 |
|---|---|
| スワイプ報告受付 | カード結果を受け取り、コイン付与 + 出現条件進行。途中離脱ボーナスを含む |
| コイン収穫 | ナマケモノが生んだ日次コインを付与。複数日蓄積にも対応 |
| 出現条件チェック（バッチ） | 条件達成済みナマケモノの到着処理。毎朝EventBridgeから実行 |

### ショップ・親密度
| 機能 | 目的 |
|---|---|
| アイテム購入 | コイン消費 → アイテム付与 |
| 贈り物 | アイテムをナマケモノに贈る → 親密度更新 → 手紙解放判定 |

### 図鑑
| 機能 | 目的 |
|---|---|
| 図鑑データ取得 | 発見状況、ヒント解放状態、条件進捗を取得 |

### 通知
| 機能 | 目的 |
|---|---|
| 夜のスワイプ通知送信 | ナマケモノ固有セリフでLINE Push通知 |
| 朝の到着通知送信 | 新しい仲間が来た朝にLINE Push通知 |
| 開発用手動トリガー | 動画撮影用に任意タイミングで通知発火。本番では無効化 |

### LINE連携
| 機能 | 目的 |
|---|---|
| Webhook受信 | LINEプラットフォームからのイベント受信 |

---

## フロントエンド 機能一覧

### 森画面
| 機能 | 目的 |
|---|---|
| 森の状態表示 | ナマケモノの配置、森の成長段階の描画 |
| 新ナマケモノ到着演出 | ログイン時に到着済みナマケモノの演出を再生 |
| コイン収穫 | 日次コインの収穫操作 |
| ナマケモノタップ | つぶやき表示（5種類、前回と重複しない） |

### スワイプ報告画面
| 機能 | 目的 |
|---|---|
| カード表示・スワイプ操作 | 10枚のダメカードを順に表示、左右スワイプで仕分け |
| 途中離脱検知 | ユーザーが「×」ボタンを押した時点で途中離脱と判定。ブラウザ直接閉じは放置ボーナス扱い |
| 結果表示 | コイン獲得演出 + ほのめかしメッセージ |

### オンボーディング
| 機能 | 目的 |
|---|---|
| 絵本表示 | 3ページの世界観導入 |
| チュートリアルスワイプ | カード1を強制右スワイプ（操作説明） |
| 初回召喚演出 | ネムリン到着の演出 |
| ショップ・贈り物誘導 | 初回購入→贈り物→親密度UPまでガイド |

---

## 備考
- 具体的なメソッドシグネチャ、APIエンドポイント設計、リクエスト/レスポンス型定義はFunctional Design + Code Generation（CONSTRUCTION phase）で詳細化
- ビジネスルール（出現条件の判定ロジック、コイン計算式、親密度の増減量）もFunctional Designで定義
