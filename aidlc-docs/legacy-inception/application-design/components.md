# コンポーネント定義 — ナマケモノの森 MVP v1

---

## フロントエンド

### 画面構成

| 画面 | 種別 | 責務 |
|---|---|---|
| **森（メイン）** | Page | ナマケモノの表示、コイン収穫、タップ操作、足跡表示 |
| **スワイプ報告** | Page | ダメカードのスワイプ、途中離脱ボーナス |
| **ショップ** | Page | アイテム一覧、購入 |
| **贈り物** | Page | ナマケモノ選択 → アイテム贈呈 → 親密度UP |
| **図鑑** | Page | シルエット/実体表示、ヒント、発見数 |
| **手紙** | Page | 手紙コレクション閲覧 |
| **オンボーディング** | Dialog（森の上にオーバーレイ） | 初回起動時のチュートリアル。絵本→スワイプ体験→召喚→ショップ→贈り物を一気通貫 |

### フロントエンドの責務範囲
- LIFF SDK連携（LINE ID取得、認証）
- 画面遷移（React Router、SPA）
- スワイプ操作のUI制御（途中離脱の「×」ボタン検知を含む）
- 演出・アニメーション（召喚演出、コイン獲得演出等）
- マスターデータの参照（カード選出のランダム化等、サーバー不要な処理）

## バックエンド

### 機能ブロック

| ブロック | 責務 |
|---|---|
| **ユーザー管理** | LINE ID登録（Push通知の前提）、ユーザー状態の取得・更新 |
| **ゲームコア** | スワイプ報告処理、コイン計算、出現条件判定、コイン収穫 |
| **ショップ・親密度** | アイテム購入、贈り物処理、親密度更新、手紙解放判定 |
| **図鑑** | 発見状況・ヒント解放状態の管理 |
| **通知** | 夜のスワイプ通知、朝の到着通知、開発用手動トリガー |
| **LINE連携** | Webhook受信、Messaging API呼び出し |

### 共通ロジック

| ロジック | 責務 |
|---|---|
| **出現条件エンジン** | ナマケモノの出現条件とユーザー履歴の照合 |
| **コイン計算** | スワイプ報酬、途中離脱ボーナス、日次収穫の計算 |
| **親密度計算** | 贈り物による親密度の増減 |
| **マスターデータ参照** | ナマケモノ定義、カード定義、アイテム定義、手紙定義 |

## インフラ

| コンポーネント | 責務 |
|---|---|
| **S3 + CloudFront** | フロントエンド（React SPA）のHTTPS配信 |
| **API Gateway + Lambda** | バックエンドAPI |
| **DynamoDB** | ユーザーデータ永続化 |
| **EventBridge** | 通知スケジューラー（夜の通知、朝の到着チェック） |
| **LINE Messaging API** | Push通知送信 |
| **LINE Login（LIFF）** | ユーザー認証 |
| **CDK** | インフラのコード管理・デプロイ |
