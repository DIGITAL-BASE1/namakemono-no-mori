## ユニットごとのストーリーカバレッジ

### Unit 1 (Backend + Infra): 全13ストーリーに関与
- すべてのストーリーがバックエンドAPIを必要とする
- 特に重要: US-01（ユーザー登録）、US-03/04（スワイプ+コイン計算）、US-05（通知）、US-07（出現条件判定）

### Unit 2 (Frontend): 全13ストーリーに関与
- すべてのストーリーがUI表示を必要とする
- 特に重要: US-01（オンボーディング全体）、US-03/04（スワイプUX）、US-12（森画面の演出）

### Unit 3 (Assets): 10ストーリーに関与
- US-04（途中離脱）、US-05（通知）、US-11（手紙）、US-13（図鑑進捗）はアセット不要
- 特に重要: US-01（絵本+ネムリン）、US-07（ナマケモノ5体）、US-12（森の背景）

## CONSTRUCTION phaseでの実行順序

Unit 1とUnit 2は**並列で**CONSTRUCTION phaseを進める。待ち時間を作らない。

```
【並列実行】
  Unit 1 Functional Design ─┐
  Unit 2 Functional Design ─┤── 同時に設計、同時に承認
                            │
  Unit 1 Infrastructure Design（Unit 2にはInfra不要）
                            │
  Unit 1 Code Generation ───┐
  Unit 2 Code Generation ───┤── 並行で実装
                            │
【結合】
  Build and Test（全ユニット結合）

Unit 3 (Assets):
  ※ CONSTRUCTION phaseの対象外。制作ガイドラインを提供し、並行で制作を進める。
  ※ 必要な時にタイムリーに納品される前提。
```
